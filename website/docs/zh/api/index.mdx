# 简介

Rspack 提供了多种界面来自定义编译过程。一些功能在不同的界面之间是重叠的，例如，一个配置选项可能可以通过命令行界面（CLI）标志设置，而其他功能可能只能通过特定界面实现。以下概念可以帮助你快速入门。

## CLI

命令行界面（CLI）用于配置和交互操作你的构建。在初期原型设计和性能分析的情况下尤其有用。大多数情况下，CLI 是用来通过配置文件和一些标志（例如 `--env`）来启动进程的。

[了解更多关于命令行界面（CLI）的信息！](/api/cli)

## Modules

Modules 接口用于定义支持的模语法和方法，详情可参考 [Modules 接口](/api/modules)。

## Hot Module Replacement 接口

Hot Module Replacement(HMR) 接口用于实现模块热替换，详情可参考 [Hot Module Replacement 接口](/api/hmr)。

## Loader 接口

Loader 用于对模块的源码进行转换，详情可参考 [Loader 接口](/api/loader-api)。

## Plugin 接口

Plugin 允许用户介入编译流程。插件可以在不同的生命周期上注册回调函数，从而修改用户的编译过程，详情可参考 [Plugin 接口](/api/plugin-api)。
